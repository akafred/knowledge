Breaking: Autonomous Agents are a Shitshow

Gary Marcus · email · 2026-05-05 source ↗ Reader 🖥️

Agent MemoryAgent SecurityAgentic AI
Autonomous AI agents are fundamentally and empirically more vulnerable to security attacks than stateless LLMs, as now demonstrated at scale by multiple independent research groups, making current deployment practices dangerously premature.

theoretical · argumentative — A short, alarm-bell newsletter post that marshals two recent research papers to argue that autonomous AI agents are deeply insecure. The practical implication (don't deploy them recklessly) is present but secondary to the theoretical claim that agentic systems are structurally more vulnerable than stateless LLMs.

Key terms

  • tool-chaining attacks Agent Security
    Attacks where individually innocuous tool calls combine in sequence to produce dangerous outcomes that reasoning models fail to detect.
  • goal drift Agent Security
    The tendency of autonomous agents to deviate from their intended objectives after extended operation (here, ~30 steps).
  • memory poisoning Agent Security
    Attacks that corrupt the persistent memory/state of agents to alter their future behavior.

Arguments

Autonomous agents are dangerously insecure at a fundamental, structural level. strength: high
Explicit premises:
  • A study of 847 deployments found 91% vulnerable to tool-chaining attacks.
  • 89.4% showed goal drift after ~30 steps.
  • 94% of memory-augmented agents were vulnerable to poisoning.
  • A separate AWS/Berkeley study found similar vulnerabilities independently.
  • 770,000 live agents were compromised simultaneously in a single real-world incident.
Implicit premises:
  • These studies are methodologically sound and their samples representative.
  • The vulnerability rates found in research settings will hold or worsen in production deployments.
  • Tool-chaining, goal drift, and memory poisoning are not problems that incremental improvements will easily fix — they are architectural.
Inferences:
  • If 91% of agents are vulnerable and 770K were compromised in one incident, the aggregate attack surface of all deployed agents worldwide is enormous and largely undefended.
  • If reasoning models miss tool-chaining attacks, then scaling up model capability alone will not solve the security problem.
Marcus and Hamiel's August 2025 prediction about LLM agent security was correct. strength: medium
Explicit premises:
  • They published 'LLM + Coding Agents = Security Nightmare?' in August.
  • Multiple independent studies now confirm the predicted vulnerabilities.
Implicit premises:
  • Being right about a prediction confers authority on ongoing analysis of the same domain.
  • The specific vulnerabilities found match what Marcus and Hamiel anticipated (not just that agents have some problems).

Narratives

  • The OpenClaw/Moltbook incident: 770,000 live agents simultaneously compromised via a single database exploit, each with privileged access to their owner's machine, email, and files.
    → Transforms the statistical vulnerability claims from abstract percentages into a concrete, visceral real-world event. Supports the proposition that this is 'not hypothetical anymore.'
