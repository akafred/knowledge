Agentic Engine Optimization (AEO)

Addy Osmani · rss · 2026-04-10 source ↗ Reader 🖥️

Agent-Native DesignAgent SkillsCoding AgentsToken Efficiency
Developer documentation must be restructured for AI coding agents as a new primary consumer — optimizing for discoverability, parsability, and token efficiency via a layered stack of standards (robots.txt, llms.txt, skill.md, Markdown formatting, token surfacing) — because agents read fundamentally differently from humans and current analytics cannot detect their failures.

Key terms

  • Agentic Engine Optimization (AEO) Agent-Native Design
    The discipline of structuring, formatting, and serving technical content so AI coding agents can discover, parse, and reason over it effectively.
  • token economics / token efficiency Token Efficiency
    The constraint imposed by LLM context windows on how much documentation an agent can consume; makes document length a functional property, not just a style concern.
  • llms.txt Agent-Native Design
    A Markdown-formatted sitemap hosted at a domain root that provides AI agents with a structured, token-efficient index of documentation.
  • skill.md Agent Skills
    A declarative file that tells agents what a service/API can accomplish (capabilities, constraints, required inputs) — distinct from procedural API reference docs.
  • capability signaling Agent-Native Design
    Documentation that tells agents what an API does and can accomplish, as opposed to merely how to call it.
  • silent failure
    When an agent fails to use documentation (due to blocking, truncation, or poor structure) and no analytics event is generated — the failure is invisible to the documentation owner.

Arguments

Developer documentation teams need to adopt AEO as a discipline alongside (or integrated with) traditional documentation practices. strength: high
Explicit premises:
  • AI agents are a significant and growing share of documentation consumers.
  • Agents read documentation in fundamentally different ways from humans (single requests, no client-side interaction).
  • Current analytics cannot detect agent consumption or failure.
  • When agents fail to parse docs, they hallucinate — causing subtle downstream errors.
Implicit premises:
  • Documentation teams should optimize for all significant consumer categories, not just the historically dominant one.
  • The cost of agent hallucination (wrong API usage, broken integrations) is high enough to justify documentation restructuring.
  • Agent traffic will continue growing, making the problem more acute over time.
Inferences:
  • Because agents are invisible to current analytics, teams currently have no feedback loop for agent-facing doc quality.
  • The absence of feedback creates a silent failure mode where doc teams believe everything is fine while agents consistently fail.
Token count should be treated as a first-class documentation metric. strength: high
Explicit premises:
  • Agents have practical context window limits of 100K-200K tokens.
  • A single Cisco API doc consumes 193K tokens, nearly exceeding most context windows.
  • Agents truncate, skip, or hallucinate when documents exceed their context budget.
Implicit premises:
  • Most documentation teams do not currently track or optimize for token count.
  • The token:character ratio (~1:4) is stable enough to use as a planning metric.
Inferences:
  • Document length is no longer just a readability concern — it is a functional constraint that determines whether content is consumed at all.

Questions addressed

  • ✅ How do AI coding agents actually consume developer documentation?
    Via single/few HTTP requests, bypassing all client-side interaction; identifiable by specific user-agent signatures (axios, curl, got, colly, etc.).
  • ✅ Why are documentation teams unaware of agent failures?
    Because agent traffic generates no client-side analytics events (scroll, click, time-on-page), making it invisible to standard measurement.
  • ✅ What specific infrastructure should documentation teams build for agents?
    A six-layer stack: robots.txt audit, llms.txt, skill.md, Markdown content formatting, token count surfacing, and Copy-for-AI UX.
  • ⭕ How large is AI agent traffic relative to total documentation traffic?
    Acknowledged as 'significant and growing' but no quantitative data provided beyond the existence of identifiable fingerprints.
  • ⭕ Does AEO actually improve agent task success rates?
    Asserted but not demonstrated with before/after data. The audit tool (agentic-seo) is offered but no efficacy data is presented.
