Anatomy of a perfect Openclaw setup

A well-configured OpenClaw workspace — organized around a set of markdown files (AGENTS.md, SOUL.md, IDENTITY.md, USER.md, HEARTBEAT.md, TOOLS.md) plus skills, memory, and secrets folders — is the key to making an AI assistant behave reliably and proactively, with AGENTS.md as the single highest-leverage file.

Key terms

  • AGENTS.md Agent Context Files (CLAUDE.md / AGENTS.md)
    The primary configuration file loaded into the system prompt at session start; defines the agent's operating directives, rules, and tool usage conventions.
  • workspace Agent Context Files (CLAUDE.md / AGENTS.md)
    The user's local configuration folder (e.g. ~/clawd/) containing all customization files, skills, memory, and secrets.
  • skill Agent Skills
    A self-contained workflow defined by a SKILL.md file with YAML frontmatter, living in its own subdirectory, that the agent invokes when a task matches the skill's description.
  • heartbeat Long-Running & Managed Agents
    A scheduled polling event where the agent checks if anything needs attention; configured via HEARTBEAT.md.
  • global install OpenClaw
    The system-level installation directory containing core docs, built-in skills, and defaults — updated but not customized by the user.

Arguments

You should invest time configuring your OpenClaw workspace rather than using defaults. strength: medium
Explicit premises:
  • Most users never open the workspace folder.
  • The workspace is the control center for agent behavior.
  • Not configuring it is a mistake.
Implicit premises:
  • An AI assistant that follows your specific instructions is more valuable than one running on defaults.
  • The time invested in configuration pays off through reduced correction and more useful output.
Inferences:
  • Clear configuration leads to better agent behavior, which leads to more productive human-agent collaboration.
AGENTS.md should be concise (under 300 lines) rather than comprehensive. strength: medium
Explicit premises:
  • Files longer than 300 lines eat too much context.
  • Instruction adherence drops with longer files.
Implicit premises:
  • LLM context windows have practical limits on how much instruction text remains effective.
  • You can link to external documentation rather than inlining it.
Inferences:
  • There is a trade-off between completeness and effectiveness in system prompts.

Questions addressed

  • ✅ What files and folders make up a complete OpenClaw workspace?
    AGENTS.md, SOUL.md, IDENTITY.md, USER.md, HEARTBEAT.md, TOOLS.md, BOOTSTRAP.md, plus skills/, memory/, .secrets/, and output/ directories.
  • ✅ What is the most important file to configure and why?
    AGENTS.md — it is loaded into the system prompt and governs all agent behavior.
  • ✅ How should you get started if setting up from scratch?
    A 6-step progression: onboard, AGENTS.md, identity files, first skill, memory setup, then TOOLS.md.
  • ✅ How does cross-session memory work?
    Daily log files in memory/ folder; agent reads today + yesterday on session start.
