DoorDash Used Copilot to Convert Its XCTest-Based iOS Test Suite to Swift Testing

Sergio De Simone · article · 2026-05-09 source ↗ Reader 🖥️

DoorDash migrated its iOS test suite from XCTest to Swift Testing at scale by combining AI-assisted code transformation (Cursor/Copilot with migration rules) with strict reliability gates (ten-consecutive-pass requirement), achieving ~60% faster CI tests while surfacing hidden concurrency defects that sequential execution had masked.

Key terms

  • Swift Testing
    Apple's modern test framework using #expect macros and @Test attributes, with native parallel execution for both sync and async code
  • migration rule
    A Cursor prompt/configuration that automates the mechanical conversion patterns from XCTest to Swift Testing (import changes, assertion rewrites, macro insertion)
  • ten-consecutive-pass rule
    A reliability gate requiring each migrated test to pass ten consecutive runs before being accepted, eliminating flakiness and timing sensitivity
  • SweetPad
    A VS Code extension providing Xcode integration, used to build targets and run tests from within the editor
  • custom MCP server MCP (Model Context Protocol)
    A locally-running server exposing an execute_vscode_command tool that lets the AI agent trigger builds and test runs directly

Questions addressed

  • ✅ How can a large iOS team migrate from XCTest to Swift Testing at scale?
    Team-owned migration with shared AI tooling (Cursor + migration rules) and strict reliability gates (ten-consecutive-pass rule)
  • ✅ What tooling and process safeguards make AI-assisted code migration reliable?
    Migration rules for mechanical conversion, custom MCP server for in-editor builds/tests, and a ten-pass gate to catch flakiness
  • ✅ What performance gains does Swift Testing offer over XCTest?
    4-7x faster in benchmarks; ~60% faster CI tests and ~40% faster builds in production
  • ✅ What hidden defects does parallel test execution surface?
    Shared state, order dependencies, and timing assumptions previously masked by sequential execution
