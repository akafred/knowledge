CNCF and Kusari Partner to Strengthen Software Supply Chain Security Across Cloud-Native Projects

Craig Risi · rss · 2026-03-23 source 🖥️

Software Supply-Chain Security
CNCF and Kusari are partnering to give cloud-native open-source projects free, AI-assisted supply-chain security tooling that embeds provenance and dependency analysis directly into developer workflows, addressing the growing complexity and attack surface of modern software supply chains.

Key terms

  • software supply chain security Software Supply-Chain Security
    The practice of ensuring integrity, provenance, and trust across all dependencies and artifacts that compose a software product
  • shift left
    Embedding security checks earlier in the development lifecycle, specifically into pull-request workflows
  • transitive dependencies Software Supply-Chain Security
    Dependencies pulled in automatically and indirectly through direct dependencies, often invisible to maintainers
  • provenance Software Supply-Chain Security
    Verifiable metadata about how, where, and by whom a software artifact was built
  • Kusari Inspector
    An AI-powered tool combining code review with dependency analysis, providing inline feedback during PRs
  • dependency confusion Software Supply-Chain Security
    An attack technique that exploits package resolution logic to substitute malicious packages for legitimate internal ones
