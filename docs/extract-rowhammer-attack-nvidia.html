Rowhammer Attack Against NVIDIA Chips

Bruce Schneier · rss · 2026-04-01 source ↗ Reader 🖥️

GPU DRAM is now a proven rowhammer attack surface: two independent research teams demonstrated that inducing bitflips in GDDR6 memory on NVIDIA Ampere GPUs can escalate to arbitrary read/write over host CPU memory and full system compromise — and a third attack works even with IOMMU enabled, removing the last easy mitigation.

Key terms

  • rowhammer
    A hardware vulnerability exploiting electrical interference in DRAM: repeatedly activating one memory row causes bit flips in adjacent rows, enabling attackers to corrupt data they should not be able to write.
  • GDDR6 / GDDR memory
    Graphics Double Data Rate memory — the DRAM used on GPUs, here shown to be susceptible to rowhammer despite its different architecture from standard DDR.
  • IOMMU
    Input-Output Memory Management Unit — a hardware mechanism that restricts DMA access from peripherals (including GPUs) to host memory; its default-disabled state is the precondition for two of the three attacks, and its bypass is the key escalation of the third.
  • page table / page directory manipulation
    The two concrete exploitation strategies: GDDRHammer corrupts last-level page tables while GeForge corrupts last-level page directories, both achieving the same outcome (arbitrary memory access) via different virtual memory structures.

Arguments

Rowhammer is a serious threat on GPUs, not just CPUs. strength: high
Explicit premises:
  • Two independent teams demonstrated bitflips in GDDR6 memory on NVIDIA Ampere GPUs.
  • Those bitflips were escalated to arbitrary read/write over CPU memory.
  • The attacks achieved full system compromise (root shell).
Implicit premises:
  • If an attack independently discovered by multiple teams achieves full system compromise, it constitutes a serious rather than theoretical threat.
Inferences:
  • GPU DRAM must now be treated as part of the system's attack surface for rowhammer-class vulnerabilities.
IOMMU is not a reliable mitigation against GPU rowhammer attacks. strength: high
Explicit premises:
  • The first two attacks require IOMMU to be disabled.
  • IOMMU is disabled by default in BIOS settings, so most systems are already exposed.
  • A third attack demonstrated root shell escalation even with IOMMU enabled.
Implicit premises:
  • A mitigation that is off by default and can be bypassed when on provides no reliable protection.
  • Defenders cannot rely on a single hardware mechanism when multiple bypass paths exist.
Inferences:
  • New mitigations beyond IOMMU are needed to address GPU rowhammer attacks.

Questions addressed

  • ✅ Can rowhammer attacks extend from CPU DRAM to GPU GDDR memory?
    Yes — demonstrated empirically on NVIDIA Ampere GPUs by three independent research teams.
  • ✅ Can GPU-side memory corruption escalate to full host system compromise?
    Yes — both GDDRHammer and GeForge achieve arbitrary CPU memory read/write and root shell.
  • ✅ Does IOMMU protect against GPU-originated rowhammer attacks?
    No — IOMMU is off by default (exposing most systems), and the third attack bypasses it even when enabled.
  • ⭕ What mitigations exist against GPU rowhammer?
    Not addressed — the post reports the attacks but does not discuss NVIDIA's response or available defenses.
