The new GPT-5.6 family: Luna, Terra, Sol

Simon Willison · email · 2026-07-10 Reader 🖥️

OpenAI's new GPT-5.6 family (Luna/Terra/Sol) leads on long-running agentic benchmarks and is pitched on token efficiency, but its benchmark story is mixed and partly self-serving and in hands-on coding it is merely competitive with Claude Fable 5 — so the more consequential news is the new API primitives (programmatic tool calling, native multi-agent, explicit prompt caching) that adopt patterns Anthropic pioneered.

The piece is practical: it exists to help a rigorous adopter decide what, if anything, changes for them.

Key terms

  • Luna / Terra / Sol
    The three sizes of the GPT-5.6 family from smallest to largest, priced (per 1M input/output tokens) at $1/$6, $2.50/$15 and $5/$30 respectively; all share a Feb 16 2026 cutoff, 1M context and 128K max output
  • Agents' Last Exam
    An evaluation of long-running professional workflows across 55 fields; OpenAI's flagship benchmark evidence, on which Sol scores 53.6
  • SWE-Bench Pro
    A self-reported coding benchmark on which Claude Fable 5 (80%) decisively beat GPT-5.6 Sol (64.6%) — and which OpenAI publicly questioned (“~30% of tasks broken”) the day before launch
  • Token efficiency (“more useful work from every token”) Token Efficiency
    OpenAI's framing of GPT-5.6's advance as getting more done per token / lower cost-per-task, not a raw-capability jump
  • Reasoning tokens
    The variable, model-specific hidden tokens spent thinking on a task; because they differ so much between models for the same task, price-per-million-tokens no longer indicates real cost
  • Programmatic Tool Calling
    A new API feature letting the model compose and run JavaScript that orchestrates tool calls; Willison speculates it could bridge MCP tools and full terminal sessions that compose CLI utilities, akin to Anthropic's dynamic filtering
  • Multi-agent (subagents in the core API) Multi-Agent Orchestration
    A new API feature letting the model spin up subagents for parallel, focused work — the sub-agent pattern now baked into the core API rather than the harness
  • Prompt cache breakpoints
    A new API feature bringing Claude-style explicit prompt-cache boundaries to OpenAI (vs OpenAI's default automatic detection, which Willison prefers)

Arguments

For a rigorous adopter, GPT-5.6 does not clearly surpass Claude Fable 5, and the release's real significance is cross-lab convergence plus new API primitives rather than a decisive capability leap. strength: medium
Explicit premises:
  • OpenAI's own headline benchmark (Agents' Last Exam) is countered by a benchmark it lost badly (SWE-Bench Pro: 64.6% vs Fable's 80%).
  • OpenAI published an audit article discrediting SWE-Bench Pro the day before launch, exactly the benchmark on which it lost.
  • In hands-on complex coding, GPT-5.6 Sol has not struck Willison as better than Fable so far.
  • The headline gain is framed as efficiency (work per token / lower cost), not raw capability.
  • Several new API features explicitly mirror Anthropic mechanisms (dynamic filtering, subagents, prompt caching).
Implicit premises:
  • Vendor-selected benchmarks are motivated evidence and should be discounted absent independent replication.
  • First-party hands-on testing on one's own hardest tasks is more decisive for an adoption verdict than aggregate benchmark scores.
  • A single reviewer's early-access sample on unspecified coding tasks generalizes enough to license a comparative verdict (the shakiest load-bearing assumption).
  • Cross-lab feature borrowing signals convergence/commoditization of the model surface rather than durable differentiation.
  • Cost claims stated as vendor 'estimated cost' and round multiples ('roughly one-quarter', 'one-sixteenth') are themselves estimates to be treated cautiously.
Inferences:
  • From mixed + self-serving benchmarks combined with hands-on parity: benchmark leadership does not equal real-world superiority.
  • From explicit feature borrowing: the frontier is converging, so the model API surface is where the interesting movement is.
Price-per-million-tokens is no longer a meaningful way to compare model cost. strength: high
Explicit premises:
  • The number of reasoning tokens spent on the same task now differs enormously between models.
Implicit premises:
  • The decision-relevant quantity is cost per completed task, not price per token.
  • Per-token price multiplied by a highly variable token count decouples the sticker price from the actual bill.
Inferences:
  • Therefore efficiency (useful work per token / cost per task) is the correct frame for comparing these models.

Narratives

  • The pelican-riding-a-bicycle probe: a full page of 18 pelicans generated across reasoning efforts (none, low, medium, high, xhigh, max) for all three models, with token counts and calculated costs — from 0.71 cents (gpt-5.6-luna at effort 'none') up to 48.55 cents (gpt-5.6-sol at max reasoning).
    → Concrete, whimsical illustration that reasoning effort — not just model size — drives cost, reinforcing the proposition that price-per-token cannot capture real cost.
  • The SWE-Bench Pro reversal: Fable 5 scored 80% to GPT-5.6 Sol's 64.6%, and OpenAI happened to publish a benchmark-audit article estimating ~30% of SWE-Bench Pro tasks are broken the day before launch.
    → Worked example that vendor benchmark presentation is strategic/self-serving — supports the 'mixed and partly self-serving' judgment.

Questions addressed

  • ✅ Is GPT-5.6 actually better than Claude Fable 5 for serious coding work?
    Partially: benchmarks are mixed (wins Agents' Last Exam, loses SWE-Bench Pro) and in hands-on use it is competent but 'not better than Fable so far' — a provisional 'no clear win' verdict.
  • ✅ Does price-per-token tell you which model is cheaper to run?
    No — because reasoning-token counts vary so much per task, per-token pricing no longer maps to real cost-per-task.
  • ✅ What is genuinely new and interesting in this release?
    The new API primitives — programmatic tool calling, native multi-agent subagents, explicit prompt-cache breakpoints, detail:original images — which Willison flags as the most interesting details.
  • ⭕ Are these new API features genuinely novel, or borrowed?
    Willison notes several mirror Anthropic (dynamic filtering, subagents, Claude-style caching) but does not ask whether OpenAI's implementations differ meaningfully or perform better — the novelty question is raised implicitly and left open.
