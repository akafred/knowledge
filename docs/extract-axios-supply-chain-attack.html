The Axios supply chain attack used individually targeted social engineering

Simon Willison · article · 2026-04-01 source ↗ Reader 🖥️

Software Supply-Chain Security
A sophisticated, individually targeted social-engineering campaign tricked an Axios maintainer into installing a Remote Access Trojan, which was then used to publish a malicious package — and every open-source maintainer of widely-used software should understand this attack pattern.

Key terms

  • supply chain attack Software Supply-Chain Security
    Compromising a widely-used software dependency at the source (maintainer credentials) so that downstream consumers unknowingly pull malicious code.
  • social engineering Software Supply-Chain Security
    A multi-step impersonation campaign — cloned company, fake Slack, staged video call — designed to build enough trust that the target installs attacker-controlled software.
  • RAT (Remote Access Trojan) Software Supply-Chain Security
    The specific malware installed during the fake meeting that gave attackers persistent access to the maintainer's machine and credentials.
  • individually targeted
    The attackers researched and tailored every element of the campaign to a single specific person, rather than casting a wide net.

Arguments

Open-source maintainers must familiarize themselves with individually-targeted social-engineering attacks. strength: high
Explicit premises:
  • The Axios attack succeeded through a sophisticated, personalized social-engineering campaign.
  • Even experienced developers (Willison himself) recognize they would likely fall for the same tactic.
  • Any maintainer of widely-used software is a worthwhile target.
Implicit premises:
  • Awareness is a meaningful (if partial) defense against social engineering.
  • The cost/benefit ratio for attackers makes this pattern repeatable and likely to recur.
  • The open-source ecosystem currently lacks structural safeguards that would make individual maintainer compromise insufficient for publishing malicious releases.
Inferences:
  • If even a security-conscious developer admits vulnerability, less-aware maintainers are at even greater risk.
  • The attack pattern described by Google (UNC1069) is being actively used against the OSS ecosystem, not just cryptocurrency/AI targets.

Narratives

Questions addressed

  • ✅ How did the Axios supply-chain attack happen?
    Through a multi-step social-engineering campaign that tricked a maintainer into installing a RAT, which then stole credentials used to publish the malicious package.
  • ⭕ What should maintainers do about this class of attack?
    The article says maintainers need to 'be familiar with this attack strategy' but does not prescribe specific defensive measures.
