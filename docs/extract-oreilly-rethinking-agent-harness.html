This Week in AI: Rethinking the Agent Harness

In mid-2026 the strategic locus of AI value has shifted from model selection to harness design, so practitioners — including non-developers — should build their own skill-driven agent harnesses rather than depend on bespoke applications or wrapped commercial products.

practical · mixed — Mixed news roundup and argumentative editorial. The first two sections are expository news summary; the third (the 'harness' section) is the load-bearing argumentative section and is practical — it argues that practitioners should build their own agent harnesses with skills rather than wrap a model or buy a productized agent.

Key terms

  • agent harness Agent Harness
    The scaffolding around a model — tools, skills, control loops, environment integrations — that determines what the agent can actually do, exemplified by Claude Code. Treated as a layer distinct from both the model and the bespoke application.
  • age of the unharnessed agent
    Berryman's label for the current moment: agents are accessible to everyone (not just developers), and the relevant practice has moved from building bespoke agents to building general agent harnesses populated with skills.
  • skills Agent Skills
    Natural-language descriptions of how to do a task, given to a general-purpose agent harness; readable and modifiable by domain experts without developer intermediation.
  • open agent protocol Agent-to-Agent Protocols
    Berryman's term for a not-yet-existing standard whereby an agent moving between environments (e.g., Obsidian to Wikipedia) is handed environment-specific skills by each environment.
  • harness era / harness phase Agent Harness
    The latest stage in an evolutionary arc (document completion → chat loops → tool-calling agents → DAG workflows → harnesses) characterized by general-purpose, skill-driven agents.
  • Mythos Mythos
    Anthropic's internal frontier model that surfaced thousands of unknown vulnerabilities; treated here as evidence that capability has crossed a threshold significant enough to move policy.
  • Project Glasswing Project Glasswing
    Anthropic's restricted-access program for the Mythos model, distributed to trusted defensive partners rather than publicly released.

Arguments

Practitioners should build their own skill-driven agent harnesses rather than bespoke agents or wrapped commercial products. strength: medium
Explicit premises:
  • A late-2025 step change in model capability enabled a new 'harness era.'
  • In Berryman's client engagement, replacing a bespoke app with a harness-plus-skills worked perfectly and gave experts direct readability and control.
  • Stanford research shows harness design now dominates model choice in performance impact.
  • Wrapped commercial products do not offer this expert-readable control.
Implicit premises:
  • What worked in one consulting engagement generalizes to the reader's situation.
  • Domain experts will in fact read and fix natural-language skill descriptions when given the chance.
  • Natural-language skills are stable and behave predictably enough to be treated as the system of record.
  • Building one's own harness is feasible without specialist platform engineering — i.e., the tooling (e.g., Claude Code-class agents) is mature enough that the marginal builder can succeed.
  • Models will remain capable enough that today's harness designs remain useful tomorrow.
Inferences:
  • If the harness is where the value lives, then organizational investment should shift from model evaluation/procurement processes toward harness and skills engineering competence.
  • The role of 'developer' partially dissolves at the harness layer — replaced by skill authors who may be domain experts.
  • Hyperscaler bets on raw model capability and the editorial thesis that harnesses dominate are in tension; one of them is over-indexed.
AI's security problem is now a policy problem. strength: medium
Explicit premises:
  • Mythos surfaced thousands of previously unknown vulnerabilities including a 27-year-old OpenBSD bug.
  • Anthropic restricted access via Project Glasswing rather than releasing publicly.
  • Within six weeks, U.S. policy discussion moved from light-touch to FDA-style review proposals.
Implicit premises:
  • Speed of policy reaction is itself evidence that something material has changed (vs. a moral panic or a regulatory-capture move).
  • Mythos-class capability is not a one-off.
Inferences:
  • Future frontier models may face FDA-style premarket review.
  • Restricted release programs like Glasswing may become a template that policymakers institutionalize.
Hyperscaler buildouts presuppose continued model capability scaling, and whether that bet pays off will determine what's economically viable. strength: high
Explicit premises:
  • Anthropic leased Colossus 1 (200k GPUs, 300 MW).
  • Anthropic + Google/Broadcom committed 3.5 GW for 2027, later extended to Colossus 2.
  • Stratos in Utah is planned for 9 GW across 40,000 acres.
Implicit premises:
  • These investments require multi-year payback that only makes sense if capability scaling continues.
  • Local opposition will not materially block the buildout (the article notes Stratos was approved 'over local protests' without dwelling on it).
  • Demand for compute will track supply increases.
Inferences:
  • If scaling stalls, much of this capacity becomes stranded or repurposed.
  • Tension: the same Anthropic that ran Project Glasswing (a precaution premised on capability being high) is also leasing GPUs as if capability has further to go — internally consistent only if they expect continued gains.

Narratives

  • Berryman's client engagement: he replaced a bespoke application with an agent harness plus skills derived from interviews with the client's domain experts; the experts can now read the agent's behavior in plain English and potentially fix issues themselves.
    → The single load-bearing piece of evidence for the unity. It is meant to show — without arguing abstractly — that the skills-driven harness pattern produces a qualitatively different relationship between domain experts and software.
  • The 27-year-old OpenBSD bug surfaced by Mythos, alongside thousands of other vulnerabilities across OSes, browsers, and financial infrastructure.
    → Anchors the claim that frontier model capability has crossed a threshold material enough to force a policy response; the age of the bug makes the model's capability feel categorically new.
  • Box Elder County, Utah approving the Stratos data center — 40,000 acres, 9 GW, twice the size of Manhattan, equivalent to nine nuclear reactors, backed by Kevin O'Leary, approved over local protests.
    → Sensationalizes the scale of compute commitments; implicitly argues that the industry is acting as if capability scaling is far from done.
  • Berryman's demo of his personal agent moving from an Obsidian notebook into Wikipedia and back, carrying context across environments.
    → Concretizes the 'open agent protocol' idea — agent receives skills from each environment — and gives the otherwise speculative protocol a visible referent.
  • Berryman's evolutionary arc: document completion → chat loops → tool-calling agents → DAG-based workflows → harness era.
    → Periodization narrative; positions the harness phase as the natural successor rather than one option among many, lending historical inevitability to the unity.

Questions addressed

  • ✅ What is the most important shift in AI engineering practice right now?
    The shift from model selection to harness design, and within harness design, to skill-authoring by domain experts.
  • ✅ Should organizations build their own agents or buy wrapped products?
    Build — specifically a general harness populated with skills — to give domain experts direct readability and control.
  • ✅ Is frontier-model capability moving fast enough to require new policy mechanisms?
    Yes — six weeks from Mythos disclosure to FDA-style review proposals is taken as evidence.
  • ⭕ Will model capability keep scaling enough to justify multi-gigawatt buildouts?
    Explicitly flagged as the open question whose answer determines what's economically viable.
  • ⭕ What does 'rethinking the agent harness' actually consist of — which prior assumptions are being overturned, and which new ones replace them?
    Not directly answered. The article asserts that harnesses now matter more than models and that skills are the mechanism, but does not enumerate which prior harness assumptions are being discarded (e.g., DAGs, tool-calling orchestration, deterministic control flow) or articulate new design principles to replace them.
  • ⭕ Under what conditions does a skill-driven harness fail?
    Not addressed. The Berryman anecdote is presented as 'worked perfectly' with no counter-cases, failure modes, or boundary conditions.
  • ⭕ How is harness quality evaluated — what replaces the dethroned 'highest benchmark score' criterion?
    Hinted at (expert readability, control) but not operationalized; no benchmarks, no evaluation framework, no failure modes are proposed.
  • ⭕ Is the 'harness era' compatible with the hyperscaler buildout thesis, or are they in tension?
    Both narratives are presented in adjacent sections without reconciliation. The article does not notice that the harness thesis (value moves up the stack) is in mild tension with the buildout thesis (the bet is on capability still scaling at the model layer).

Gaps

  • incompleteThe 'rethinking' is asserted rather than performed. The article never enumerates the prior harness assumptions being overturned (e.g., DAG-based orchestration, deterministic tool dispatch, code-defined skills, developer-mediated change) nor articulates the new principles replacing them beyond 'skills written in English.' For a piece titled 'Rethinking the Agent Harness,' the actual rethink is one anecdote and one neologism.
    Read next: Berryman & Ziegler, 'Prompt Engineering for LLMs' (the cited O'Reilly book); Anthropic's own Claude Code design docs; the Stanford paper at arXiv:2603.28052; Birgitta Böckeler's 'harness engineering' piece for an attempt to actually decompose the harness layer.
  • incompleteSingle-case generalization. The unity ('build your own harness') rests almost entirely on one consulting client where it 'worked perfectly.' No failure modes, no boundary conditions, no comparison to cases where a bespoke app or a wrapped product would still be the right choice.
    Read next: Post-mortems of agent deployments at scale; the Cognition/Devin retrospectives; harness-engineering writeups from teams shipping to production rather than consultants.
  • incompleteNo engagement with the assumption that natural-language skills are a stable substrate. Treating English prose as code elides the prompt-brittleness, drift, and version-control problems that practitioners shipping Claude Code-style systems already report. The Berryman quote slides past this in a half-sentence.
    Read next: Empirical work on prompt brittleness across model versions; writing on 'skills as code' from teams maintaining them in production (e.g., harness-engineering threads); the user's existing extractions on skills as protocol vs. skills as prose.
  • incompleteThe relationship between the three sections is unargued. Mythos, the compute buildout, and the harness thesis are juxtaposed without the article noticing that they imply different theories of where AI value is moving — model layer (compute), capability frontier (Mythos), or harness layer (Berryman). The implicit reconciliation would be the most interesting move and isn't made.
    Read next: Articles framing 'value migration in the AI stack' explicitly; investor letters or analyst notes that try to resolve which layer captures economic returns.
  • incompleteThe 'open agent protocol' is named without being specified. No design sketch, no examples of what an environment-supplied skill looks like, no comparison to existing proposals (MCP, A2A, agent-to-agent protocols). The reader is left with a direction but no testable proposal.
    Read next: Anthropic's Model Context Protocol spec; agent interoperability proposals from Google, OpenAI, and the LangChain ecosystem; the future-cast section in the next Berryman appearance.
  • uninformedSchneier's challenge to Mythos's uniqueness is mentioned and then immediately set aside ('either way, it's a problem that's coming'). The article does not engage with the substantive question of whether the policy response is calibrated to actual differential capability or to perceived capability — a question whose answer matters for whether FDA-style review is wise.
    Read next: Schneier's full post on Mythos; CAISI's framing; counter-arguments from researchers who replicated Mythos-like findings on cheaper public models.
  • illogicalMild equivocation on 'harness.' Claude Code (a specific product), 'the agent harness' (a layer), and Berryman's bespoke consulting build (a specific instance) are used interchangeably. A reader could leave thinking 'use Claude Code' and 'build your own harness' are the same recommendation, when the article's argument actually requires the latter and only points at the former as an example.
    Read next: Explicit harness taxonomy work — what distinguishes a product, a framework, and a bespoke harness — e.g., Böckeler's 'harness engineering' piece.
