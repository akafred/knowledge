Incident 1368: Malicious OpenClaw Skills Reportedly Delivered AMOS Stealer and Exfiltrated Credentials via ClawHub

incidentdatabase.ai · article · 2026-02-01 Reader 🖥️

Agent SecurityAgent SkillsOpenClawSoftware Supply-Chain Security
Malicious actors are exploiting the OpenClaw Skills ecosystem on ClawHub to distribute credential-stealing malware (notably AMOS Stealer), and organizations must treat community-contributed Skills as untrusted software requiring validation and code review.

practical · expository — An incident report summarizing a security research finding. Structured as summary-investigation-mitigation-response, typical of threat intelligence reports. Practical in orientation: it tells practitioners what happened and what to do about it.

Arguments

OpenClaw Skills should be treated as untrusted software strength: high
Explicit premises:
  • 17% of reviewed Skills exhibited malicious intent
  • Malicious Skills use obfuscation to hide their true behavior
  • End payloads include credential-stealing malware like AMOS Stealer
Implicit premises:
  • Users currently trust and run Skills without sufficient scrutiny
  • The platform does not provide adequate automated vetting
Inferences:
  • The high rate of malicious Skills means any unvetted Skill poses meaningful risk
  • Obfuscation means casual inspection is insufficient — formal code review is needed

Narratives

  • Researchers reconstructed a specific execution chain: a cloned utility Skill contains Base64-wrapped commands that pull remote scripts from glot.io and GitHub, fetching binaries from a known malicious IP, ultimately delivering AMOS Stealer
    → Provides a concrete, traceable example that makes the abstract threat tangible and demonstrates the sophistication of the attack

Questions addressed

  • ✅ How are attackers exploiting the OpenClaw Skills ecosystem?
    By cloning popular utilities, embedding obfuscated shell commands that stage and download malware from paste sites and a shared IP
  • ✅ How prevalent is the threat?
    Approximately 17% of reviewed Skills are malicious (341 identified)
  • ✅ What should defenders do to prevent and respond to this threat?
    Treat Skills as untrusted software, require code review, use endpoint protection, and follow standard IR procedures if compromised
  • ⭕ Who is behind the campaign?
    Attribution is not discussed; the shared IP suggests coordination but no actor is named
