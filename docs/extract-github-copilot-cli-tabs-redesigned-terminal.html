GitHub Copilot CLI Gets Tabs and No-Config-File Tool Setup in Redesigned Terminal UI

Mark Silvester · rss · 2026-07-10 Reader 🖥️

Agent SkillsAgent Tool ManagementCoding AgentsDeveloper Experience & FlowMCP (Model Context Protocol)
GitHub has made Copilot CLI's redesigned terminal interface generally available, adding tabbed in-terminal browsing of GitHub issues, pull requests and gists, guided in-session configuration of MCP servers, skills and plugins that removes the need to hand-edit config files, and accessibility-focused theming — consolidating the terminal into a self-contained, extensible surface for agentic development.

practical · expository — A short product-update news report describing a shipped release of GitHub Copilot CLI's redesigned terminal interface. Almost entirely expository (feature enumeration), but it carries an implicit argumentative undertone about where agentic-CLI tooling is heading: the terminal absorbing IDE-grade UX and treating skills/MCP/plugins as first-class, config-free extension surfaces.

Key terms

  • Copilot CLI / agentic CLI
    GitHub's command-line coding agent that works with GitHub directly from the terminal, now with a full terminal UI (TUI) layered over the agentic loop.
  • redesigned terminal interface (TUI)
    A tabbed, mouse-and-keyboard, theme-aware, responsive text UI rendered inside the terminal — effectively an IDE-like workbench in a shell.
  • MCP servers / GitHub MCP Registry
    Model Context Protocol servers that extend the CLI's tools; the GitHub MCP Registry is a browseable catalogue from which servers are installed in-session (via /mcp search) and become available immediately without a restart.
  • skills Agent Skills
    Individually toggleable capability modules managed via the /skills command.
  • plugins
    Installable extensions sourced from a marketplace, a repository, or a local path via the /plugin command.
  • no-config-file / in-session configuration
    A guided, interactive-form workflow for adding tools and changing settings that removes the need to hand-edit configuration files.
  • theme-aware semantic colours / screen-reader support
    Accessibility affordances: colour roles that adapt to the terminal theme, selectable colour modes (/theme), and auto-enabled screen-reader mode with labelled icons and disabled animations.

Arguments

The redesign makes Copilot CLI a more capable, more self-contained developer surface. strength: medium
Explicit premises:
  • Tabs bring repo-scoped Issues, Pull requests and Gists into the session with keyboard/mouse actions to reference, open, or search them.
  • In-session guided forms replace hand-editing of configuration files for MCP servers, skills, plugins and settings.
  • Accessibility and readability improvements (theming, responsive layout, screen-reader support) broaden who can use it.
Implicit premises:
  • Developers prefer to stay in the terminal, so reducing context switches to the browser improves flow and productivity.
  • Lower setup friction increases adoption of the CLI's extension mechanisms.
  • A terminal can host IDE-grade UX (tabs, settings dialogs, object browsing) without losing its speed/simplicity advantages.
Inferences:
  • Consolidating GitHub browsing, tool configuration and theming into the CLI therefore raises its value relative to editing config files and switching to a browser.
An agentic CLI should natively support MCP servers, skills and plugins as configurable, discoverable, config-file-free surfaces. strength: medium
Explicit premises:
  • The CLI now ships /mcp add, /mcp search (GitHub MCP Registry), /skills, /plugin and /settings as in-session commands.
  • Newly added MCP servers are available immediately without restarting the CLI.
Implicit premises:
  • Skills/MCP/plugins are the standard extension model for coding agents.
  • Hand-editing configuration files is a friction barrier worth engineering away.
  • A curated registry is the right discovery mechanism for agent tools.
Inferences:
  • Therefore making these extension surfaces first-class, in-session and registry-installable is the correct direction for the tool.
