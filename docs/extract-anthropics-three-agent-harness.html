Anthropic's Designs Three-Agent Harness Supports Long-Running Full-Stack AI Development

Leela Kumili · rss · 2026-04-06 source ↗ Reader 🖥️

Agent HarnessContext Rot & Context LimitsLong-Running & Managed AgentsMulti-Agent OrchestrationRiding Model Improvements
Splitting long-running AI development tasks among three specialized agents — planner, generator, and evaluator — with structured handoff artifacts solves the two core failure modes of autonomous coding sessions (context loss and self-evaluation bias), producing higher-quality and more consistent outputs.

practical · expository — A journalistic summary of an Anthropic engineering blog post. Describes an engineering pattern and its rationale. Practical: argues that practitioners should adopt this harness design. Notably thin on original analysis — the real text is the Anthropic engineering post, not this summary.

Key terms

  • harness Agent Harness
    A structured multi-agent workflow that coordinates distinct agents (planner, generator, evaluator) through defined handoffs — the scaffolding around the AI, not the AI itself
  • context reset Context Rot & Context Limits
    Deliberately clearing the agent's context at a defined boundary and handing over a structured artifact (state summary, progress record) so the next agent starts fresh from a known state — as opposed to compaction, which attempts to preserve continuity
  • evaluator agent Multi-Agent Orchestration
    A separate AI agent whose sole job is to judge the generator's output using predefined criteria and few-shot examples; it navigates live pages and provides detailed critiques rather than passively reviewing text
  • compaction Context Rot & Context Limits
    A technique for preserving context by summarizing or compressing prior conversation, allowing the model to appear to have longer memory — contrasted with context resets because it causes models to grow cautious as limits approach

Arguments

A separate evaluator agent is necessary for quality control in long-running AI development strength: medium
Explicit premises:
  • AI agents overrate their own outputs, especially on subjective tasks
  • Self-evaluation fails on design tasks because agents cannot reliably judge their own work
Implicit premises:
  • Separation of concerns (producer vs. assessor) is a reliable quality principle borrowed from human engineering practice
  • Calibration through few-shot examples is sufficient to make an evaluator agent reliable — its own potential biases are not addressed
Inferences:
  • The evaluator pattern could apply to any AI task where output quality is hard to specify in advance — not just frontend design
  • The article does not address whether the evaluator itself needs evaluation — a regress it assumes away
Context resets with structured handoff artifacts outperform compaction for long-running sessions strength: low
Explicit premises:
  • Compaction makes models cautious as they approach context limits, potentially affecting performance
  • Context loss causes task failure or incoherence in long sessions
Implicit premises:
  • A well-designed handoff artifact captures sufficient state to resume the task — this assumption bears all the weight but is not examined
  • The caution induced by compaction is a meaningful performance degradation, not a minor artifact
Inferences:
  • The design quality of the handoff artifact is the real bottleneck — the article doesn't address what must be in it
  • Context reset + handoff is itself a form of state management, implying the agent system needs a formal state model for the task being performed

Narratives

  • Anthropic's frontend design pipeline: evaluator agent navigates live pages using Playwright MCP, interacts with the interface, scores on four criteria (design quality, originality, craft, functionality), and provides critiques that drive 5-15 iteration cycles per run, sometimes taking up to four hours
    → Demonstrates that the evaluator agent pattern produces 'designs that combine visual distinction with functional accuracy' — the primary evidence claim for the three-agent design
  • Artem Bredikhin's LinkedIn observation: 'every new context window is amnesia. The breakthrough is structure: JSON feature specs, enforced testing, commit-by-commit progress, and an init script that ensures every session starts with a working app'
    → External validation of the article's central claim — structured artifacts solve the amnesia problem. Also reveals what a handoff artifact might concretely contain (JSON specs, tests, git state, init script)

Questions addressed

  • ✅ How can AI agents maintain coherence and make progress over multi-hour sessions without context degradation?
    Context resets with structured handoff artifacts allow each session to start from a defined, known state
  • ✅ How can AI agents produce reliable evaluations of their own output on subjective tasks?
    Separate evaluator agent calibrated with few-shot examples and scoring criteria
  • ⭕ What specifically must a handoff artifact contain to enable coherent session continuation?
  • ⭕ How should evaluation criteria be derived and validated for a given task type?
