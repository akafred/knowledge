A model that produces code which compiles and passes the tests it was given is not the same as a model that produces robust code

Key terms

Arguments

Claims like 'AI writes 80% of our code' are misleading because they conflate test-passing with robustness; realism about the mechanism (next-word prediction) is necessary for genuine progress.
Explicit premises:
  • Code that compiles and passes tests is not the same as robust code
  • Next-word prediction goes far in writing code but falls short on robustness
Implicit premises:
  • Throughput metrics (like '80% of code') measure generation volume, not quality
  • Robustness requires reasoning about edge cases that next-word prediction does not reliably provide
Inferences:
  • Therefore vendor claims about AI-written code percentages are misleading without robustness evidence
  • Therefore inexperienced coders relying on AI are most exposed to this gap
