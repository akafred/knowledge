Key terms

  • Design at the Speed of AI Non-Engineers Building
    UI design iteration that matches the generation speed of large language/image models — essentially instantaneous from prompt to artifact
  • Stitch Non-Engineers Building
    Google's AI tool that 'stitches' together the design and code layers — from text description to visual design to exportable frontend code in one flow
  • Hatter
    An AI agent within Stitch that handles multi-step design workflows autonomously (named, presumably, after the Mad Hatter — creative, unpredictable, generative)
  • MCP export MCP (Model Context Protocol)
    Native Model Context Protocol export — allowing Stitch outputs to be consumed by AI coding agents and other MCP-compatible tools

Arguments

You should use Stitch instead of traditional design workflows strength: medium
Explicit premises:
  • Traditional mockups take a week
  • Stitch generates editable designs plus real code from text descriptions
  • The tool is free
Implicit premises:
  • Speed of iteration matters more than precision of initial specification
  • AI-generated UI quality is sufficient for production use
  • The design-development handoff (not the design itself) is the primary bottleneck in product development
  • Founders and PMs are better served by generating many quick variants than by crafting one careful mockup
Inferences:
  • If design cost (time + skill + money) approaches zero, then design ceases to be a scarce resource and becomes an abundant one
  • If design is abundant, the bottleneck shifts from 'can we make this look good' to 'do we know what to build'
AI agents (Hatter) can handle multi-step design workflows, not just single-shot generation strength: low
Explicit premises:
  • Stitch introduces Hatter, an agent for multi-step design tasks
Implicit premises:
  • Single-shot generation is insufficient for real design work, which involves iteration, revision, and context-dependent decisions
  • An agentic architecture (autonomous multi-step reasoning) is the right abstraction for design workflows
Inferences:
  • The trajectory is from tool (human-directed) to agent (AI-directed with human oversight)

Narratives

  • The 'napkin to production' narrative: a founder has an idea, sketches it on a napkin, and seconds later has a working UI with real code — no designer, no developer, no waiting
    → Compresses the entire product development pipeline into a single magical step, creating urgency and desire by contrasting with the painful status quo
  • The 'wasted week' narrative: the current design process is wasteful, slow, and a drag on founders who should be moving fast
    → Creates a pain point (waste/friction) that positions Stitch as the obvious remedy
