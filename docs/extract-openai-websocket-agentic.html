OpenAI Introduces Websocket-Based Execution Mode to Reduce Latency in Agentic Workflows

Leela Kumili · rss · 2026-04-20 source ↗ Reader 🖥️

Agentic LoopInference Optimization
Transport-layer optimization — replacing per-step HTTP round-trips with a persistent WebSocket connection — yields substantial latency reductions (30-40%) in agentic AI workflows, demonstrating that system-level infrastructure changes outside the model itself are now a primary lever for improving real-world AI performance.

practical · reportorial — A trade-press news report on a concrete engineering change (HTTP to WebSocket) in OpenAI's Responses API. Reportorial rather than expository — announces the change, quotes adopters, and presents benchmark numbers without developing an argument or framework of its own.

Key terms

  • WebSocket-based execution mode
    A persistent, bidirectional connection between client and server for the OpenAI Responses API that replaces per-step HTTP requests with a long-lived session supporting continuous data exchange
  • agentic workflows Agentic Loop
    Multi-step AI reasoning processes involving tool calls, intermediate reasoning, and follow-up queries — used in coding agents and real-time AI systems
  • transport layer (in agentic systems)
    The communication patterns and connection management infrastructure between client and LLM server, as distinct from the model itself
  • cold starts
    The latency penalty of establishing new HTTP connections for each step in a multi-tool agent chain

Arguments

Transport-layer optimization is a high-leverage intervention for agentic AI performance. strength: medium
Explicit premises:
  • Inference speeds have improved, making network round-trips the dominant latency source.
  • WebSocket connections eliminate repeated handshakes and enable streaming.
  • OpenAI reports up to 40% latency reduction in production.
  • Independent adopters (Vercel, Cline, Cursor) report 30-40% improvements.
Implicit premises:
  • The per-step HTTP pattern was not questioned earlier because model inference time dominated total latency.
  • Latency reduction at the transport layer compounds across multi-step workflows — each saved round-trip multiplies by the number of steps.
Inferences:
  • Therefore, as models get faster, the non-model infrastructure becomes the primary optimization target.
