Spec-Driven Development

Workflows where human-authored specifications are the primary artifact and AI implements against them — with live debate over whether specs replace or complement reading the code.

Also known as: spec-driven development, specs-to-code, specs-to-code (the movement he opposes), PRD (product requirements document), PRD, AI PRD, destination document / journey document, sedimentary spec, grill-me (skill), Grill Me, spec-anchored, amnesiac AI

Documents (6)

  • [FULL WORKSHOP] AI Coding For Real Engineers - Matt Pocock, AI Hero AI Engineer (channel); speaker: Matt Pocock (AI Hero) · 2026-04-24 source 🖥️
    AI is not a new paradigm for software engineering — the fundamentals that make working with humans effective (small tasks, vertical slices, deep modules, fast feedback loops, alignment through dialogue, TDD) are exactly what make working with AI agents effective; the path from idea to shipped feature is therefore a deliberately planned, human-driven pipeline (align → PRD → vertical-slice Kanban → AFK implementation → QA) in which the codebase itself is treated as the battleground, not the spec.
  • It Ain't Broke: Why Software Fundamentals Matter More Than Ever Matt Pocock (AI Hero conference talk, AI Engineer) · 2026-04-23 source 🖥️
    Because AI amplifies the cost of bad code through software entropy, investing in software design fundamentals — shared understanding, ubiquitous language, deep modules, TDD, and deliberate architecture — matters more in the AI era than it ever did before.
  • Sam Hatoum's Post Sam Hatoum · 2026-04-20 source 🖥️
    Spec-driven development with AI is not waterfall because waterfall's fatal flaw was slow feedback, not specifications themselves, and AI collapses the feedback loop from months to minutes — making specs more essential, not less.
  • Ashutosh Singh's Post Ashutosh Singh · 2026-04-10 source 🖥️
    Traditional product specs fail for AI features because they lack the technical specificity engineers need; PMs must replace vague feature descriptions with precise behavioral definitions, golden examples, failure-mode handling, cost/latency constraints, explicit scope boundaries, and eval criteria.
  • No, AI Did Not Kill Agile Andrea Laforgia · 2026-04-00 source 🖥️
    AI accelerates building software but does not reduce the epistemic uncertainty about what to build, so the iterative learning loop at the core of Agile becomes more important, not less, when coding is nearly free.
  • Presentation: AI-First Software Delivery: Balancing Innovation with Proven Practices Wes Reisz · 2025-01-01 source 🖥️
    AI-first software delivery is not one-size-fits-all: teams should choose between supervised and unsupervised agentic approaches based on code longevity and their ability to verify outputs automatically, using structured human-AI collaboration (RIPER-5) to preserve engineering discipline — because AI amplifies whatever practices you already have, good or bad.
