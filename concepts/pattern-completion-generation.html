Pattern-Completion Generation (Not Reasoning)

LLMs generate by statistically continuing patterns — from training data or from whatever sits in the live context window — rather than by reasoning, so output is steered as much by what surrounds the model (a codebase, a prompt, prior examples) as by explicit instruction.

Also known as: next-word prediction, mimicry, bounded patterns, in-context learning, agents amplify what they see

Part of: LLM Limitations

Documents (4)

  • 11 Tips For AI Coding With Ralph Wiggum Matt Pocock · 2026-07-11 source ↗ Reader 🖥️
    Autonomous agent loops ("Ralph") become reliably productive only when the human stops writing code and instead wraps the loop in disciplined scaffolding — an explicit end-state/stop-condition, an externalized progress file, commit-blocking feedback loops, small steps, risk-first sequencing, stated quality expectations and a sandbox — scaffolding that compensates for the agent's lack of judgment and cross-session memory and that, unguardrailed, would let the loop accelerate software entropy.
  • Richard Dawkins and The Claude Delusion Gary Marcus (Marcus on AI, Substack) · 2026-05-09 Reader 🖥️
    Dawkins' claim that Claude may be conscious is a textbook Argument from Personal Incredulity — the very fallacy he once famously ridiculed — because it infers internal states from output behavior while ignoring that LLMs are mimics whose fluent language is produced by statistical pattern-matching over training data, not by felt experience or genuine understanding.
  • A model that produces code which compiles and passes the tests it was given is not the same as a model that produces robust code Gary Marcus · 2026-05-09 Reader 🖥️
    Code that compiles and passes its given tests is not the same as robust code; conflating the two inflates AI-coding claims, and progress requires acknowledging this gap rather than celebrating throughput.
  • Philosophy Eats AI: What Leaders Should Know MIT Sloan Management Review · 2025-05-14 source ↗ Reader 🖥️
    Because generative AI is built from and operates through philosophical constructs (purposes, categories, knowledge), the executives who win with AI will be those who deliberately map and reconcile their organization's teleology, ontology, and epistemology — not those who optimize ethics, metrics, or models in isolation.
