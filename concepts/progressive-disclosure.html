Progressive Disclosure

Loading only the instructions relevant to the current task into an agent's context — minimal root files with detail pulled in on demand, protecting the finite instruction-following budget.

Also known as: progressive disclosure, instruction budget, push vs. pull (for coding standards)

Part of: Context Engineering

Documents (3)

  • Your AGENTS.md might be a big problem Matt Pocock (AI Hero) · 2026-07-03 Reader 🖥️
    A reactively bloated AGENTS.md degrades an AI coding agent because it spends the model's finite instruction-following budget on every request, so you should keep the root file to 3-5 essentials and push everything else into nested files the agent loads only when relevant.
  • Agent Skills Addy Osmani · 2026-05-09 source ↗ Reader 🖥️
    AI coding agents default to the shortest path to 'done' and systematically skip the invisible senior-engineer work (specs, tests, reviews, scope discipline) that separates reliable software from code that breaks — skills are portable, workflow-shaped markdown files that bolt that scaffolding back on by making it impossible for the agent to rationalize skipping it.
  • [FULL WORKSHOP] AI Coding For Real Engineers - Matt Pocock, AI Hero AI Engineer (channel); speaker: Matt Pocock (AI Hero) · 2026-04-24 source ↗ Reader 🖥️
    AI is not a new paradigm for software engineering — the fundamentals that make working with humans effective (small tasks, vertical slices, deep modules, fast feedback loops, alignment through dialogue, TDD) are exactly what make working with AI agents effective; the path from idea to shipped feature is therefore a deliberately planned, human-driven pipeline (align → PRD → vertical-slice Kanban → AFK implementation → QA) in which the codebase itself is treated as the battleground, not the spec.
