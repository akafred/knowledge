Scope Discipline

Constraining agents (and AI-accelerated teams) to touch only what was asked — countering over-editing, opportunistic rewrites and AI-fueled scope creep.

Also known as: scope discipline, agent drift, Agent drift, Over-Editing, over-editing, scope creep, Scope Creep Kraken, tentacle, tool-driven momentum, anti-rationalization table

Documents (5)

  • Agent Skills Addy Osmani · 2026-05-09 source ↗ Reader 🖥️
    AI coding agents default to the shortest path to 'done' and systematically skip the invisible senior-engineer work (specs, tests, reviews, scope discipline) that separates reliable software from code that breaks — skills are portable, workflow-shaped markdown files that bolt that scaffolding back on by making it impossible for the agent to rationalize skipping it.
  • Accelerating Delivery: How AI Agents Can Own the Initial Code Draft Stephen Walker · 2026-04-20 source ↗ Reader 🖥️
    Autonomous AI agent workflows can reliably automate the journey from a Jira ticket to a first pull request if teams wrap inherently probabilistic LLMs in deterministic guardrails (branch scoping, hooks, orchestration) and multi-stage adversarial review loops that enforce architectural standards before human eyes ever see the code.
  • Coding Models Are Doing Too Much nrehiew.github.io · 2026-04-20 source ↗ Reader 🖥️
    Current coding models systematically rewrite code beyond the minimal fix required — a widespread, measurable failure mode called over-editing — and reinforcement learning can train models to produce faithful minimal edits without degrading their general coding ability.
  • Meet the Scope Creep Kraken Tim O'Brien · 2026-04-16 Reader 🖥️
    AI-assisted development breaks the natural friction that once constrained scope creep by making feature generation so fast that demonstrations replace deliberate decisions, and the only remedy is to reassert traditional project discipline — written scope, explicit decision-making, and downstream-impact assessment — over the seductive momentum of tool-driven output.
  • Ashutosh Singh's Post Ashutosh Singh · 2026-04-10 Reader 🖥️
    Traditional product specs fail for AI features because they lack the technical specificity engineers need; PMs must replace vague feature descriptions with precise behavioral definitions, golden examples, failure-mode handling, cost/latency constraints, explicit scope boundaries, and eval criteria.
